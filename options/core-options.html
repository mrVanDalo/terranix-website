--- title: core-options letter: c info: "" ---

These options are delivered by the terranix core. They are always available.

  • backend.etcd

    type: null or submodule

    default: null

    description: etcd backend https://www.terraform.io/docs/backends/types/etcd.html

    defined: in modules/terraform/backends.nix

  • backend.etcd.endpoints

    type: string

    default: null

    description: A space-separated list of the etcd endpoints

    defined: in modules/terraform/backends.nix

  • backend.etcd.password

    type: null or string

    default: null

    description: the password

    defined: in modules/terraform/backends.nix

  • backend.etcd.path

    type: string

    default: null

    description: The path where to store the state

    defined: in modules/terraform/backends.nix

  • backend.etcd.username

    type: null or string

    default: null

    description: the username

    defined: in modules/terraform/backends.nix

  • backend.local

    type: null or submodule

    default: null

    description: local backend https://www.terraform.io/docs/backends/types/local.html

    defined: in modules/terraform/backends.nix

  • backend.local.path

    type: string

    default: null

    description: path to the state file

    defined: in modules/terraform/backends.nix

  • backend.s3

    type: null or submodule

    default: null

    description: s3 backend https://www.terraform.io/docs/backends/types/s3.html

    defined: in modules/terraform/backends.nix

  • backend.s3.bucket

    type: string

    default: null

    description: bucket name

    defined: in modules/terraform/backends.nix

  • backend.s3.key

    type: string

    default: null

    description: path to the state file in the bucket

    defined: in modules/terraform/backends.nix

  • backend.s3.region

    type: string

    default: null

    description: region of the bucket

    defined: in modules/terraform/backends.nix

  • data

    type:

    default: {}

    description: Data objects, are queries to use resources which are already exist, as if they are created by a the resource option. See for more details : https://www.terraform.io/docs/configuration/data-sources.html

    defined: in core/terraform-options.nix

    example: {}

  • locals

    type:

    default: {}

    description: Define terraform variables with file scope. Like modules this is terraform intern and terranix has better ways. See for more details : https://www.terraform.io/docs/configuration/locals.html

    defined: in core/terraform-options.nix

    example: {"locals":{"owner":"Community Team","service_name":"forum"}}

  • module

    type:

    default: {}

    description: A terraform module, to define multiple resources, for sharing or duplication. The terraform module system, and has nothing to do with the module system of terranix or nixos. See for more details : https://www.terraform.io/docs/configuration/modules.html

    defined: in core/terraform-options.nix

    example: {"module":{"consul":{"source":"github.com/hashicorp/example"}}}

  • output

    type:

    default: {}

    description: Useful in combination with terraform_remote_state. See for more details : https://www.terraform.io/docs/configuration/outputs.html

    defined: in core/terraform-options.nix

    example: {"output":{"instance_ip_addr":{"value":"aws_instance.server.private_ip"}}}

  • provider

    type:

    default: {}

    description: Define you API connection. Don't use secrets in here, they will be visible in the nix-store and the resulting config.tf.json. Instead use terraform variables. See for more details : https://www.terraform.io/docs/configuration/providers.html or https://www.terraform.io/docs/providers/index.html

    defined: in core/terraform-options.nix

    example: {"provider":{"google":{"project":"acme-app","region":"us-central1"}}}

  • provisioner.privateKeyFile

    type: string

    default: null

    description: PrivateKey for provisioning via ssh access see https://www.terraform.io/docs/provisioners/connection.html This is an agnostic option, option-authors should use this options to implement server provisioning.

    defined: in modules/provisioner.nix

    example: ~/.ssh/id_rsa

  • remote_state.etcd

    type: attribute set of submodules

    default: {}

    description: etcd remote state https://www.terraform.io/docs/backends/types/etcd.html

    defined: in modules/terraform/backends.nix

  • remote_state.etcd.<name>.endpoints

    type: string

    default: null

    description: A space-separated list of the etcd endpoints

    defined: in modules/terraform/backends.nix

  • remote_state.etcd.<name>.password

    type: null or string

    default: null

    description: the password

    defined: in modules/terraform/backends.nix

  • remote_state.etcd.<name>.path

    type: string

    default: null

    description: The path where to store the state

    defined: in modules/terraform/backends.nix

  • remote_state.etcd.<name>.username

    type: null or string

    default: null

    description: the username

    defined: in modules/terraform/backends.nix

  • remote_state.local

    type: attribute set of submodules

    default: {}

    description: local remote state https://www.terraform.io/docs/backends/types/local.html

    defined: in modules/terraform/backends.nix

  • remote_state.local.<name>.path

    type: string

    default: null

    description: path to the state file

    defined: in modules/terraform/backends.nix

  • remote_state.s3

    type: attribute set of submodules

    default: {}

    description: s3 remote state https://www.terraform.io/docs/backends/types/s3.html

    defined: in modules/terraform/backends.nix

  • remote_state.s3.<name>.bucket

    type: string

    default: null

    description: bucket name

    defined: in modules/terraform/backends.nix

  • remote_state.s3.<name>.key

    type: string

    default: null

    description: path to the state file in the bucket

    defined: in modules/terraform/backends.nix

  • remote_state.s3.<name>.region

    type: string

    default: null

    description: region of the bucket

    defined: in modules/terraform/backends.nix

  • resource

    type:

    default: {}

    description: The backbone of terraform and terranix to change and create state. See for more details : https://www.terraform.io/docs/configuration/resources.html

    defined: in core/terraform-options.nix

    example: {"resource":{"aws_instance":{"web":{"ami":"ami-a1b2c3d4","instance_type":"t2.micro"}}}}

  • terraform

    type:

    default: {}

    description: Terraform configuration. But for backends have a look at the terranix options backend.etcd, backend.local and backend.s3. See for more details : https://www.terraform.io/docs/configuration/terraform.html

    defined: in core/terraform-options.nix

    example: {"terraform":{"backend":{"s3":{"bucket":"mybucket","key":"path/to/my/key","region":"us-east-1"}}}}

  • users

    type: attribute set of attribute set of submoduless

    default: {}

    description: User management. `users.group.username` is the path. All members in the `admins` group should be able to ssh to servers. This is an agnostic option, option-authors should use this options to implement server provisioning.

    defined: in modules/users.nix

    example: {"admins":{"lassuls":{"publicKey":"ssh-rsa ABKAB3NzaC1yc2EAAAA.."},"mrVanDalo":{"publicKey":"ssh-rsa AAAAB3NzaC1yc2EAAAA.."}}}

  • users.<name>.<name>.publicKey

    type: null or string

    default: null

    description: ssh public key of user

    defined: in modules/users.nix

    example: ${ file( ~/.ssh/id_rsa.pub ) }

  • variable

    type:

    default: {}

    description: Input Variables, which can be set by `--var=name` or by environment variables prefixt with `TF_VAR_`. Usually used in terraform modules or to ask for API tokens. See for more details : https://www.terraform.io/docs/configuration/variables.html

    defined: in core/terraform-options.nix

    example: {"variable":{"image_id":{"description":"The id of the machine image (AMI) to use for the server.","type":"string"}}}
