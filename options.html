Here are the options negatively available by terranix. You can also seem them via man terranix-modules.

This page is still a work in progress and will change soon

backend.etcd
etcd backend https://www.terraform.io/docs/backends/types/etcd.html
backend.etcd.endpoints
A space-separated list of the etcd endpoints
backend.etcd.password
the password
backend.etcd.path
The path where to store the state
backend.etcd.username
the username
backend.local
local backend https://www.terraform.io/docs/backends/types/local.html
backend.local.path
path to the state file
backend.s3
s3 backend https://www.terraform.io/docs/backends/types/s3.html
backend.s3.bucket
bucket name
backend.s3.key
path to the state file in the bucket
backend.s3.region
region of the bucket
cloudflare.enable
Whether to enable enable cloudflare provider.
cloudflare.provider
This option has no description.
cloudflare.provider.api_client_logging
Whether to print logs from the API client (using the default log library logger). This can also be specified with the CLOUDFLARE_API_CLIENT_LOGGING shell environment variable.
cloudflare.provider.email
The email associated with the account. This can also be specified with the CLOUDFLARE_EMAIL shell environment variable.
cloudflare.provider.max_backoff
Maximum backoff period in seconds after failed API calls This can also be specified with the CLOUDFLARE_MAX_BACKOFF shell environment variable.
cloudflare.provider.min_backoff
Minimum backoff period in seconds after failed API calls. This can also be specified with the CLOUDFLARE_MIN_BACKOFF shell environment variable.
cloudflare.provider.org_id
Configure API client with this organisation ID, so calls use the organization API rather than the (default) user API. This is required for other users in your organization to have access to the resources you manage. This can also be specified with the CLOUDFLARE_ORG_ID shell environment variable.
cloudflare.provider.retries
Maximum number of retries to perform when an API request fails. This can also be specified with the CLOUDFLARE_RETRIES shell environment variable.
cloudflare.provider.rps
RPS limit to apply when making calls to the API. This can also be specified with the CLOUDFLARE_RPS shell environment variable.
cloudflare.provider.token
The Cloudflare API token. This can also be specified with the CLOUDFLARE_TOKEN shell environment variable.
cloudflare.provider.use_org_from_zone
Takes a zone name value. This is used to lookup the organization ID that owns this zone, which will be used to configure the API client. If org_id is also specified, this field will be ignored. This can also be specified with the CLOUDFLARE_ORG_ZONE shell environment variable.
github.enable
Whether to enable enable github provider.
github.provider
This option has no description.
github.provider.token
login token
hcloud.enable
Whether to enable enable hcloud provider.
hcloud.provider
This option has no description.
hcloud.provider.endpoint
- (Optional, string) Hetzner Cloud API endpoint, can be used to override the default API Endpoint https://api.hetzner.cloud/v1.
hcloud.provider.poll_interval
- (Optional, string) Configures the interval in which actions are polled by the client. Default 500ms. Increase this interval if you run into rate limiting errors.
hcloud.provider.token
To get started using the API you first need an API token. Sign in into the Hetzner Cloud Console choose a project, go to Access → Tokens, and create a new token. Make sure to copy the token because it won’t be shown to you again. A token is bound to a project, to interact with the API of another project you have to create a new token inside the project
provisioner.privateKeyFile
private key filename for ssh access for provioning see https://www.terraform.io/docs/provisioners/connection.html
remote_state.etcd
etcd remote state https://www.terraform.io/docs/backends/types/etcd.html
remote_state.etcd.<name>.endpoints
A space-separated list of the etcd endpoints
remote_state.etcd.<name>.password
the password
remote_state.etcd.<name>.path
The path where to store the state
remote_state.etcd.<name>.username
the username
remote_state.local
local remote state https://www.terraform.io/docs/backends/types/local.html
remote_state.local.<name>.path
path to the state file
remote_state.s3
s3 remote state https://www.terraform.io/docs/backends/types/s3.html
remote_state.s3.<name>.bucket
bucket name
remote_state.s3.<name>.key
path to the state file in the bucket
remote_state.s3.<name>.region
region of the bucket
users
user managment for servers. usually admins. A gloabal container of users which servers should pull from and allow access to them.
users.<name>.<name>.publicKey
public ssh key of user.
