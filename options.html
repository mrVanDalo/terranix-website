  • cloudflare.enable

    type: boolean

    default: false

    description: Whether to enable enable cloudflare provider.

    defined: in modules/provider/cloudflare.nix

    example: true

  • cloudflare.provider

    type: submodule

    default: {"email":"${ var.cloudflare_email }","token":"${ var.cloudflare_token }"}

    description: This option has no description.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.api_client_logging

    type: null or boolean

    default: null

    description: Whether to print logs from the API client (using the default log library logger). This can also be specified with the CLOUDFLARE_API_CLIENT_LOGGING shell environment variable.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.email

    type: string

    default: ${ var.cloudflare_email }

    description: The email associated with the account. This can also be specified with the CLOUDFLARE_EMAIL shell environment variable.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.max_backoff

    type: null or signed integer

    default: null

    description: Maximum backoff period in seconds after failed API calls This can also be specified with the CLOUDFLARE_MAX_BACKOFF shell environment variable.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.min_backoff

    type: null or signed integer

    default: null

    description: Minimum backoff period in seconds after failed API calls. This can also be specified with the CLOUDFLARE_MIN_BACKOFF shell environment variable.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.org_id

    type: null or string

    default: null

    description: Configure API client with this organisation ID, so calls use the organization API rather than the (default) user API. This is required for other users in your organization to have access to the resources you manage. This can also be specified with the CLOUDFLARE_ORG_ID shell environment variable.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.retries

    type: null or string

    default: null

    description: Maximum number of retries to perform when an API request fails. This can also be specified with the CLOUDFLARE_RETRIES shell environment variable.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.rps

    type: null or signed integer

    default: null

    description: RPS limit to apply when making calls to the API. This can also be specified with the CLOUDFLARE_RPS shell environment variable.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.token

    type: string

    default: ${ var.cloudflare_token }

    description: The Cloudflare API token. This can also be specified with the CLOUDFLARE_TOKEN shell environment variable.

    defined: in modules/provider/cloudflare.nix

  • cloudflare.provider.use_org_from_zone

    type: null or string

    default: null

    description: Takes a zone name value. This is used to lookup the organization ID that owns this zone, which will be used to configure the API client. If org_id is also specified, this field will be ignored. This can also be specified with the CLOUDFLARE_ORG_ZONE shell environment variable.

    defined: in modules/provider/cloudflare.nix

    type: string

Make sure to copy the token because it won’t be shown to you again. A token is bound to a project, to interact with the API of another project you have to create a new token inside the project
